# Continuity Principles

## Context Is A Project Asset

Projects lose momentum when the reason behind decisions disappears.

Operational rule: preserve goals, assumptions, decisions, current state, and known issues in dedicated files.

## Memory Should Be Lightweight

The memory system should be easy enough to update at the end of a tired work session.

Operational rule: write brief entries that help the next session start faster.

## Preserve Intent, Not Just Artifacts

Code and docs show what exists. Memory explains what was tried, what matters, and what changed.

Operational rule: update memory when the project direction or understanding changes.

## Make Handoff Boring

A project is healthier when a new person can continue without a meeting.

Operational rule: current state, open questions, and known issues should be readable before handoff.

## AI Tools Need Context, Too

AI-assisted workflows improve when assistants inherit real project context instead of generic prompts.

Operational rule: point assistants at foundation and memory files before asking for implementation.
