# Definition Of Done

A task is done when it is complete, understandable, and durable.

## Required For Most Changes

- [ ] The change solves the stated problem.
- [ ] The implementation follows project rules and code standards.
- [ ] Relevant tests were added or updated.
- [ ] Existing tests pass or failures are documented.
- [ ] Documentation changed when behavior, setup, or usage changed.
- [ ] Memory files changed when decisions, assumptions, or current state changed.
- [ ] No unrelated refactors or formatting churn were included.
- [ ] Known follow-up work is captured in roadmap, issues, or memory.

## Required For User-Facing Changes

- [ ] The README or usage docs explain the new behavior.
- [ ] The changelog captures the impact.
- [ ] Edge cases and failure states were reviewed.

## Required For Architecture Changes

- [ ] The architecture guide was updated.
- [ ] The decision log records the tradeoff.
- [ ] Migration or rollback notes exist if needed.
