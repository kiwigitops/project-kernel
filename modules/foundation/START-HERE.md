# Foundation Module

Use this module to establish the baseline rules for a project: how files are organized, how code should feel, what quality means, and what must be true before work is considered done.

## Read First

1. [`PROJECT-RULES.md`](PROJECT-RULES.md)
2. [`SETUP-CHECKLIST.md`](SETUP-CHECKLIST.md)
3. [`CODE-STANDARDS.md`](CODE-STANDARDS.md)
4. [`ARCHITECTURE-GUIDE.md`](ARCHITECTURE-GUIDE.md)
5. [`PROJECT-TYPE-PROFILES.md`](PROJECT-TYPE-PROFILES.md)
6. [`DEFINITION-OF-DONE.md`](DEFINITION-OF-DONE.md)

## What To Fill In Immediately

- Project name and purpose.
- Supported runtimes, platforms, or environments.
- Required commands for setup, test, lint, build, and release.
- Folder ownership and naming conventions.
- The quality bar for merging work.

## How This Connects To Other Modules

- Pair with [`../docs`](../docs/START-HERE.md) to explain the project publicly.
- Pair with [`../memory`](../memory/START-HERE.md) to preserve decisions and current state.
- Pair with [`../workflow`](../workflow/START-HERE.md) to turn the standards into a working loop.
