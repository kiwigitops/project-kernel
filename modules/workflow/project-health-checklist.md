# Project Health Checklist

Use this checklist during maintenance, before handoff, or before a major release.

## Documentation Health

- [ ] README explains what the project does and who it is for.
- [ ] Setup instructions work from a clean environment.
- [ ] Architecture notes match the current system.
- [ ] Roadmap reflects current priorities.
- [ ] Changelog captures user-visible changes.

## Memory Health

- [ ] Current state is up to date.
- [ ] Decision log includes recent tradeoffs.
- [ ] Known issues include impact and workarounds.
- [ ] Open questions have owners or next actions.
- [ ] Session notes explain how to resume.
- [ ] Design review notes capture recent UX/UI, design system, or retention findings when relevant.

## Engineering Health

- [ ] Common commands are documented.
- [ ] Tests cover critical behavior.
- [ ] Dependency choices are intentional.
- [ ] Fragile areas are documented.
- [ ] Definition of done still fits the project.

## Workflow Health

- [ ] Planning, review, and release flows are clear.
- [ ] Contributors know how to make safe changes.
- [ ] Maintenance work is visible.
- [ ] Handoff can happen without private context.
