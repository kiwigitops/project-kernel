# Planning Workflow

## Purpose

Turn a problem or idea into scoped work.

## Steps

1. Define the user or maintainer problem.
2. State the smallest useful outcome.
3. Identify constraints and non-goals.
4. Check current priorities and open questions.
5. List affected modules, docs, and tests.
6. Break work into reviewable pieces.
7. Decide what must be true for the work to be done.

## Output

- Problem statement.
- Scope.
- Non-goals.
- Implementation outline.
- Test and documentation expectations.
