# Review Workflow

## Purpose

Catch bugs, unclear decisions, and maintainability risks before work ships.

## Steps

1. Read the intent of the change.
2. Check behavior against the stated problem.
3. Check edge cases, failures, and security concerns.
4. Check whether tests prove the important behavior.
5. Check docs, changelog, and memory updates.
6. Separate blocking findings from suggestions.
7. Record follow-up work instead of stretching the current change.

## Output

- Review findings.
- Required changes.
- Test gaps.
- Approval or next action.
