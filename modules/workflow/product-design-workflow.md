# Product Design Workflow

Use this workflow when a project needs product, UX, UI, or retention decisions before or during implementation.

## Flow

```text
Frame the product risk -> Choose the right skill -> Produce evidence -> Update memory -> Implement or defer
```

## Choose A Skill

| Situation | Skill |
| --- | --- |
| Risky product idea needs validation | [`../skills/design-sprint/SKILL.md`](../skills/design-sprint/SKILL.md) |
| Project needs tokens, components, or a design system | [`../skills/ui-ux-system-generator/SKILL.md`](../skills/ui-ux-system-generator/SKILL.md) |
| iOS app or SwiftUI/UIKit screen needs platform review | [`../skills/ios-platform-design/SKILL.md`](../skills/ios-platform-design/SKILL.md) |
| Users are confused or tasks fail | [`../skills/usability-audit/SKILL.md`](../skills/usability-audit/SKILL.md) |
| Users sign up but do not return | [`../skills/retention-hook-analysis/SKILL.md`](../skills/retention-hook-analysis/SKILL.md) |
| UI works but looks inconsistent or unpolished | [`../skills/ui-refactor-visual-audit/SKILL.md`](../skills/ui-refactor-visual-audit/SKILL.md) |

## Process

1. State the product or design risk.
2. Pick the narrowest skill that fits.
3. Gather required inputs.
4. Run the skill and produce its expected output.
5. Record decisions, issues, assumptions, and open questions in memory.
6. Convert accepted recommendations into implementation tasks.
7. Use review workflow before merge or release.

## Definition Of Ready For Implementation

- [ ] The user or product problem is clear.
- [ ] The skill output includes specific recommendations.
- [ ] Critical risks are recorded.
- [ ] Accepted decisions are recorded.
- [ ] Follow-up implementation work is scoped.
