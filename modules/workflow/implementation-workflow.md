# Implementation Workflow

## Purpose

Build changes in a way that stays understandable and reviewable.

## Steps

1. Read the relevant foundation, memory, and architecture notes.
2. Inspect existing patterns before adding new ones.
3. Make the smallest coherent change.
4. Add or update tests near the changed behavior.
5. Run focused checks early.
6. Update docs or memory as soon as decisions appear.
7. Keep unrelated cleanup out of the change unless it is required.

## Output

- Implemented change.
- Tests or verification notes.
- Updated docs or memory where needed.
- Known follow-up work.
