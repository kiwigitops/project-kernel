# Release Workflow

## Purpose

Ship changes with clear scope, verification, and communication.

## Steps

1. Define release scope.
2. Confirm all required checks pass.
3. Review changelog and migration notes.
4. Check known issues and open questions.
5. Confirm docs match released behavior.
6. Tag or publish according to project practice.
7. Record release notes and follow-up work.

## Output

- Release version or identifier.
- Changelog entry.
- Verification summary.
- Known risks and follow-ups.
