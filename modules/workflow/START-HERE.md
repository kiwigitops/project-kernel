# Workflow Module

Use this module to define how work moves through the project: planning, implementation, review, documentation, issue triage, release, and maintenance.

## Core Loop

```text
Plan -> Build -> Review -> Document -> Ship -> Preserve Context
```

## Update Rhythm

- Use planning workflow before non-trivial work.
- Use implementation workflow while building.
- Use review workflow before merge.
- Use documentation workflow whenever behavior or setup changes.
- Use maintenance workflow on a recurring cadence.
