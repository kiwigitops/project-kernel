# Maintenance Workflow

## Purpose

Keep the project healthy after the first version works.

## Cadence

Choose a cadence that matches project activity:

- weekly for active projects,
- monthly for stable projects,
- before every handoff,
- before every release.

## Steps

1. Review known issues.
2. Review open questions.
3. Review dependencies and tooling.
4. Remove stale docs or update them.
5. Check whether architecture notes still match reality.
6. Update current state and priorities.
7. Capture cleanup work with clear value.

## Output

- Updated project state.
- Prioritized maintenance tasks.
- Removed stale context.
- Clear next actions.
