# Contributing

Thank you for considering a contribution.

## Project Expectations

- Keep changes focused.
- Explain tradeoffs in the pull request.
- Add or update tests for behavior changes.
- Update docs when usage, setup, or architecture changes.
- Record important decisions when the change affects project direction.

## Local Setup

See [`SETUP.md`](SETUP.md).

## Development Flow

1. Create or choose an issue.
2. Confirm the expected behavior.
3. Make the smallest coherent change.
4. Run checks.
5. Update documentation and changelog when relevant.
6. Open a pull request with context and verification steps.

## Pull Request Checklist

- [ ] The change is scoped and explained.
- [ ] Tests were added or updated.
- [ ] Documentation was updated when needed.
- [ ] The changelog was updated for user-visible changes.
- [ ] Known follow-up work is captured.

## Code Of Conduct

Be direct, kind, and practical. Assume good intent, resolve ambiguity early, and keep the project maintainable for the next person.
