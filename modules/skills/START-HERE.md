## Integrated Sourced Skills

These skills are curated from real public Claude skill implementations and normalized into the Project Kernel directory format:

| Skill | Use |
| --- | --- |
| [`design-sprint`](design-sprint/SKILL.md) | Validate risky product ideas with a prototype and user tests |
| [`ui-ux-system-generator`](ui-ux-system-generator/SKILL.md) | Generate or extract tokens, components, states, and design system rules |
| [`ios-platform-design`](ios-platform-design/SKILL.md) | Design or review iOS/iPadOS interfaces against platform conventions |
| [`usability-audit`](usability-audit/SKILL.md) | Run heuristic, task, accessibility, and dark-pattern usability audits |
| [`retention-hook-analysis`](retention-hook-analysis/SKILL.md) | Diagnose activation, retention, habit loops, prompts, and ethical engagement |
| [`ui-refactor-visual-audit`](ui-refactor-visual-audit/SKILL.md) | Improve visual hierarchy, spacing, typography, color, tokens, and UI polish |

Each integrated skill includes `SKILL.md`, `checklist.md`, and templates where useful.

## How To Use A Skill

1. Open the skill when the matching work begins.
2. Follow the process in order unless the project context justifies a change.
3. Produce the expected output.
4. Update related docs or memory files.
