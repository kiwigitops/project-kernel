# Usability Audit Checklist

## Orientation

- [ ] App or site identity is clear.
- [ ] Page or screen title is clear.
- [ ] Current location is visible.
- [ ] Navigation options are understandable.
- [ ] Primary action is obvious.

## Task Completion

- [ ] User can start the task without explanation.
- [ ] Each step builds confidence.
- [ ] Required information is visible when needed.
- [ ] Optional and required fields are clear.
- [ ] Completion state is visible.

## Heuristics

- [ ] System status is visible.
- [ ] Language matches user vocabulary.
- [ ] Users can undo, cancel, or go back.
- [ ] Patterns are consistent.
- [ ] Errors are prevented where possible.
- [ ] Options are shown instead of memorized.
- [ ] Expert shortcuts do not clutter novice paths.
- [ ] UI avoids unnecessary noise.
- [ ] Error messages explain cause and fix.
- [ ] Help is contextual and findable.

## Accessibility

- [ ] Text contrast meets WCAG AA.
- [ ] Interactive elements have labels.
- [ ] Keyboard navigation works.
- [ ] Focus states are visible.
- [ ] Touch targets are large enough.
- [ ] Meaning is not color-only.
- [ ] Critical information is not hover-only.

## Ethics

- [ ] No hidden costs.
- [ ] No confirmshaming.
- [ ] No pre-checked consent.
- [ ] Cancellation and opt-out are not buried.
- [ ] CTA hierarchy does not manipulate against user interest.

## Memory

- [ ] P0/P1 issues added to known issues.
- [ ] Open research questions captured.
- [ ] Audit summary added to design review notes.
