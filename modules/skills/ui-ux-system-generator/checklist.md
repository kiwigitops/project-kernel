# UI/UX System Generator Checklist

## Product Fit

- [ ] Surface type is identified.
- [ ] Primary user task is stated.
- [ ] Visual tone matches product context.
- [ ] System does not overfit one page.

## Tokens

- [ ] Color primitives exist.
- [ ] Semantic color tokens exist.
- [ ] Text tokens exist.
- [ ] Surface and border tokens exist.
- [ ] Spacing scale uses consistent increments.
- [ ] Radius scale is constrained.
- [ ] Shadow/elevation scale is named.
- [ ] Motion tokens include duration and easing.
- [ ] Dark mode remaps semantic tokens.

## Components

- [ ] Each component has purpose.
- [ ] Variants are limited and named.
- [ ] Default, hover, focus, active, disabled states are covered.
- [ ] Loading, empty, error, and success states are covered where relevant.
- [ ] Accessibility labels and keyboard behavior are specified.
- [ ] Responsive behavior is specified.
- [ ] Content rules are specified.

## Implementation

- [ ] Components use semantic HTML.
- [ ] Reusable patterns are extracted.
- [ ] Hardcoded visual values are avoided.
- [ ] Tailwind/shadcn usage follows project conventions if applicable.
- [ ] No mixed icon families.
- [ ] No one-off styling unless explicitly justified.

## Memory

- [ ] Major design decisions added to decision log.
- [ ] Open questions captured.
- [ ] Design system status added to design review notes.
