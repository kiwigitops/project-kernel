# UI Refactor Visual Audit Checklist

## Hierarchy

- [ ] One primary action is clear.
- [ ] Primary content is emphasized.
- [ ] Secondary information is de-emphasized.
- [ ] Labels do not overpower values.
- [ ] Page can be understood with a quick scan.

## Spacing And Layout

- [ ] Spacing uses a consistent scale.
- [ ] Related elements are close.
- [ ] Unrelated groups are separated.
- [ ] Content width is constrained.
- [ ] Forms are not unnecessarily full-width.
- [ ] Decorative wrappers are removed.

## Typography

- [ ] Type scale is consistent.
- [ ] Headings have tight line height.
- [ ] Body text has comfortable line height.
- [ ] Font weights are purposeful.
- [ ] Text does not overflow on mobile.

## Color And Tokens

- [ ] Text colors have clear primary/secondary/tertiary roles.
- [ ] Surface and border colors are tokenized.
- [ ] Semantic feedback colors exist.
- [ ] Contrast meets WCAG AA.
- [ ] Dark mode uses semantic remapping.
- [ ] Color is not the only signal.

## Components And States

- [ ] Components reuse variants.
- [ ] Hover states exist where relevant.
- [ ] Focus states are visible.
- [ ] Disabled states are understandable.
- [ ] Loading states are stable.
- [ ] Empty and error states are designed.
- [ ] Icons use a consistent family.

## Motion And Depth

- [ ] Shadows communicate elevation.
- [ ] Motion explains state or hierarchy.
- [ ] Reduced-motion is supported.
- [ ] Layout does not jump during loading.

## Memory

- [ ] Token decisions added to decision log.
- [ ] Visual issues added to design review notes.
- [ ] Implementation constraints captured.
