# Documentation Skill

## What It Is For

Use this skill to create or update documentation that helps someone act.

## When To Use It

- Setup changes.
- Public behavior changes.
- Architecture changes.
- A decision needs to be preserved.
- A repeated question appears.

## Process

1. Identify the reader and the action they need to take.
2. Choose the smallest document that answers the question.
3. Start with the practical path, then add context.
4. Include commands, file paths, examples, or decision links where useful.
5. Remove outdated guidance.
6. Cross-link related docs.
7. Verify that a new reader can follow the doc without private context.

## Expected Output

- Updated doc.
- Related links.
- Clear next action for the reader.

## Pitfalls To Avoid

- Writing project marketing where setup instructions are needed.
- Keeping dead placeholders.
- Documenting implementation trivia that will rot quickly.
- Splitting related context across too many files.
