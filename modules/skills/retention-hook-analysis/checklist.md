# Retention Hook Analysis Checklist

## Behavior Definition

- [ ] Target behavior is specific.
- [ ] Frequency is defined.
- [ ] User segment is defined.
- [ ] Metric is behavior-based.

## B=MAP

- [ ] Motivation is identified.
- [ ] Ability is rated.
- [ ] Prompt is identified.
- [ ] Prompt timing is justified.
- [ ] Behavior is above the action line.

## Ability Chain

- [ ] Time friction rated.
- [ ] Money friction rated.
- [ ] Physical effort rated.
- [ ] Mental effort rated.
- [ ] Social deviance rated.
- [ ] Non-routine friction rated.
- [ ] Weakest link identified.
- [ ] Starter step defined.

## Hook Model

- [ ] External trigger identified.
- [ ] Internal trigger hypothesis identified.
- [ ] Core action is simple.
- [ ] Reward is valuable and variable.
- [ ] Investment follows reward.
- [ ] Investment improves next use.

## Habit Testing

- [ ] Habitual user criteria defined.
- [ ] Habitual user rate measured or planned.
- [ ] Habit path identified.
- [ ] Aha moment hypothesis identified.
- [ ] Cohort tracking defined.

## Ethics

- [ ] Product improves user life or work.
- [ ] Vulnerable populations considered.
- [ ] Notifications are controllable.
- [ ] No manufactured anxiety.
- [ ] Users can leave with their data when relevant.

## Memory

- [ ] Hypothesis added to assumptions.
- [ ] Experiment added to decision log or roadmap.
- [ ] Metrics added to current state.
- [ ] Open data questions captured.
