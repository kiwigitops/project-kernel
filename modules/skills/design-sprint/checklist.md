# Design Sprint Checklist

## Pre-Sprint

- [ ] Big challenge is important enough for a sprint.
- [ ] Decision maker is present or available.
- [ ] Team includes product, design, engineering, customer insight, and business context.
- [ ] Calendar is blocked.
- [ ] Target users are recruitable.
- [ ] Prototype tool is chosen.
- [ ] Test sessions are scheduled or recruitment path is active.

## Monday: Map

- [ ] Long-term goal written.
- [ ] Sprint questions written as risks.
- [ ] Customer journey map has 5-15 steps.
- [ ] Expert interviews completed.
- [ ] HMW notes captured.
- [ ] HMW notes voted.
- [ ] Target customer and target moment selected.

## Tuesday: Sketch

- [ ] Lightning demos captured.
- [ ] Team chose swarm or divide.
- [ ] Notes completed silently.
- [ ] Rough ideas completed.
- [ ] Crazy 8s completed.
- [ ] Solution sketches are anonymous and self-explanatory.

## Wednesday: Decide

- [ ] Sketches reviewed silently.
- [ ] Heat map voting completed.
- [ ] Straw poll completed.
- [ ] Decision maker chose direction.
- [ ] Storyboard has 10-15 steps.
- [ ] Prototype scope is small enough for one day.

## Thursday: Prototype

- [ ] Prototype roles assigned.
- [ ] Prototype only covers tested flow.
- [ ] Copy is realistic.
- [ ] Interview script drafted.
- [ ] Internal trial run completed.
- [ ] Prototype is believable but not overbuilt.

## Friday: Test

- [ ] Participants match target user.
- [ ] Interviewer avoids leading.
- [ ] Observers capture behavior and quotes.
- [ ] Patterns are separated from one-off reactions.
- [ ] Decision is made from evidence.
- [ ] Memory files updated.
