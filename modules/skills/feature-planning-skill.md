# Feature Planning Skill

## What It Is For

Use this skill to turn an idea into a scoped, implementable change.

## When To Use It

- A feature request is vague.
- A change touches multiple modules.
- The project needs a plan before implementation.
- The work could expand without boundaries.

## Process

1. State the user problem.
2. Define the smallest useful outcome.
3. List constraints and non-goals.
4. Identify affected files, modules, or systems.
5. Decide what needs tests or docs.
6. Break implementation into reviewable steps.
7. Capture open questions before coding.

## Expected Output

- Problem statement.
- Scope and non-goals.
- Implementation outline.
- Test and documentation plan.
- Open questions.

## Pitfalls To Avoid

- Planning implementation before defining success.
- Letting nice-to-have work into the first version.
- Ignoring migration or compatibility.
- Skipping documentation impact.
