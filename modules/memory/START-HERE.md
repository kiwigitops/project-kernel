# Memory Module

Use this module to preserve project context over time. It is especially useful for solo developers returning after a pause, teams onboarding new contributors, and AI-assisted coding sessions that need durable context.

## Read First

1. [`PROJECT-CONTEXT.template.md`](PROJECT-CONTEXT.template.md)
2. [`CURRENT-STATE.template.md`](CURRENT-STATE.template.md)
3. [`DECISION-LOG.template.md`](DECISION-LOG.template.md)
4. [`WORKING-ASSUMPTIONS.template.md`](WORKING-ASSUMPTIONS.template.md)
5. [`SESSION-NOTES.template.md`](SESSION-NOTES.template.md)
6. [`ONBOARDING-CHECKLIST.template.md`](ONBOARDING-CHECKLIST.template.md)

## Update Rhythm

- Start of project: fill context, constraints, assumptions, and priorities.
- Before a work session: read current state and open questions.
- During work: add decisions and implementation notes.
- End of session: update session notes, known issues, and current state.
- Before handoff: make sure a new person can continue from memory alone.

## What Belongs Here

- Why the project exists.
- What is currently true.
- Decisions and tradeoffs.
- Constraints and assumptions.
- Known issues and open questions.
- Implementation notes that are not public docs yet.

## What Does Not Belong Here

- Secrets.
- Credentials.
- Personal notes unrelated to project work.
- Large logs that should live in issue trackers or observability tools.
