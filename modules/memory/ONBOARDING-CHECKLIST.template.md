# Onboarding Checklist

Use this checklist when a new contributor, teammate, maintainer, or AI assistant starts working on the project.

## First Read

- [ ] README.
- [ ] Setup guide.
- [ ] Architecture notes.
- [ ] Project rules.
- [ ] Current state.
- [ ] Known issues.
- [ ] Open questions.

## First Run

- [ ] Install dependencies.
- [ ] Configure environment variables.
- [ ] Run tests.
- [ ] Run the project locally.
- [ ] Run lint or build checks.

## First Change

- [ ] Choose a small issue or task.
- [ ] Read related implementation notes.
- [ ] Confirm expected behavior.
- [ ] Make a scoped change.
- [ ] Run relevant checks.
- [ ] Update docs or memory if context changed.

## Onboarding Notes

- [What was confusing?]
- [What should be documented better?]
- [What should the next contributor know?]
