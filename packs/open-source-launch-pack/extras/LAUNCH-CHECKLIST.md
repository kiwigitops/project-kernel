# Open-Source Launch Checklist

Use this before sharing a repo publicly.

## Repository Basics

- [ ] README explains the project in one sentence.
- [ ] Quick start works from a clean clone.
- [ ] License is present.
- [ ] Contribution rules are clear.
- [ ] Roadmap shows current direction.
- [ ] Changelog exists.

## Contributor Readiness

- [ ] Setup instructions list required tools.
- [ ] Common commands are documented.
- [ ] Issues or roadmap identify good next work.
- [ ] Review expectations are clear.

## Release Readiness

- [ ] Versioning approach is documented.
- [ ] Release workflow is documented.
- [ ] Known limitations are visible.
- [ ] Security or support expectations are stated when relevant.
