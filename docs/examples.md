# Examples

These are common ways to adopt Project Kernel.

## Solo Developer Starting A New App

Use:

- `modules/foundation`
- `modules/docs`
- `modules/memory`
- `modules/workflow`

First actions:

1. Fill project purpose and success criteria.
2. Add setup commands.
3. Write current state.
4. Create the first roadmap slice.
5. End each session with session notes.

## Open-Source Repo Launch

Use:

- `modules/foundation`
- `modules/docs`
- `modules/workflow`
- selected `modules/skills`

First actions:

1. Polish README.
2. Add contributing rules.
3. Start changelog.
4. Write roadmap.
5. Add issue triage and release workflow.

## AI-Assisted Coding Workflow

Use:

- `modules/foundation`
- `modules/memory`
- `modules/skills`
- `modules/workflow`

First actions:

1. Point the assistant at project rules and current state.
2. Keep decisions in the decision log.
3. Use the review skill before accepting generated changes.
4. Update session notes before stopping.

## Team Handoff Setup

Use:

- `modules/docs`
- `modules/memory`
- `modules/workflow`

First actions:

1. Fill architecture summary.
2. Update current state.
3. List open questions.
4. Capture known issues.
5. Write next best actions.
