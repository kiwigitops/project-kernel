# Philosophy

Project Kernel is built on one idea: projects should preserve intent, not just files.

## Projects Lose Context Fast

The first few days of a project create decisions that shape everything later. Those decisions often live in memory, chats, or commit messages that nobody reads. Project Kernel creates obvious places for that context before it disappears.

## Structure Should Be Useful Immediately

This project avoids ornamental process. A file should help someone start, decide, debug, review, ship, or maintain.

## Good Defaults Beat Perfect Systems

The modules are intentionally simple Markdown. They work in any stack and can be edited by anyone. Automation exists to reduce copying friction, not to turn Project Kernel into a framework.

## AI-Assisted Work Needs More Context, Not More Magic

AI coding workflows improve when assistants can read project rules, current state, decision logs, and implementation notes. The memory and skills modules are designed to make that context durable.

## The Best Repos Onboard Fast

A strong project explains itself. A new contributor should quickly understand what the project is, how to run it, why it is shaped this way, what is currently happening, and how to make a safe change.
