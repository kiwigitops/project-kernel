# Examples

These examples show how Project Kernel looks when applied to real project situations.

## Included Examples

- [`seeded-empty-project`](seeded-empty-project/README.md): before and after for a blank project.
- [`solo-app-example`](solo-app-example/README.md): solo developer starting a new app.
- [`ai-assisted-project`](ai-assisted-project/README.md): AI-assisted coding workflow with memory.
- [`open-source-repo-example`](open-source-repo-example/README.md): public repo launch setup.
- [`team-handoff-example`](team-handoff-example/README.md): onboarding and ownership transfer.

## How To Read These

Each example lists:

- starting situation,
- modules or pack used,
- resulting folder shape,
- first actions,
- what changes after adoption.
