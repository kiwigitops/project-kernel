# Team Handoff Example

## Starting Situation

A maintainer needs to hand a project to another person or team without relying on meetings and private memory.

## Use

```bash
node scripts/install-pack.js packs/team-handoff-pack ../handoff-project
```

## Result

The target project receives:

- setup and architecture templates,
- current state and known issues,
- decision log and open questions,
- review and maintenance workflow,
- handoff checklist.

## First Actions

1. Make setup steps current.
2. Explain architecture in plain language.
3. Update current state.
4. List known issues and workarounds.
5. Capture open questions and owners.

## Handoff Outcome

The next maintainer can run the project, understand the current shape, see risks, and choose the next useful action.
