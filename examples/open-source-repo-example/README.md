# Open-Source Repo Example

## Starting Situation

A project is ready to become public, but the README, contribution guide, changelog, and roadmap are thin.

## Use

```bash
node scripts/install-pack.js packs/open-source-launch-pack ../public-repo
```

## Result

The repo receives:

- public README structure,
- setup and contribution templates,
- changelog and roadmap structure,
- issue triage and release workflows,
- launch checklist.

## First Actions

1. Write the one-sentence value proposition.
2. Add a quick start that works from a clean clone.
3. Fill contribution expectations.
4. Start changelog and roadmap.
5. Complete launch checklist.

## What Improves

- Visitors understand the repo faster.
- Contributors know how to help.
- Releases become easier to communicate.
- Project direction is visible.
