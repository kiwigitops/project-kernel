# Solo App Example

## Starting Situation

A solo developer is starting a personal app and expects to work on it across irregular sessions.

## Use

```bash
node scripts/install-pack.js packs/solo-dev-os ../my-app
```

## Result

The app receives:

- foundation rules,
- documentation templates,
- memory files,
- workflow loops,
- project philosophy,
- solo cadence extras.

## First Actions

1. Fill `memory/PROJECT-CONTEXT.template.md`.
2. Fill `memory/CURRENT-STATE.template.md`.
3. Fill `memory/PRIORITIES.template.md`.
4. Fill setup commands.
5. Write the first session note.

## Before Project Kernel

The developer remembers goals mentally and returns later to a cold project.

## After Project Kernel

The developer resumes from current state, priorities, known issues, and next best actions.
